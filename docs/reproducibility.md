# OHIE Reproducibility Scorecard

Phase 6 classifies every benchmark by whether another researcher can rerun it from repository-local inputs and documented assumptions.

## Status Definitions

| Status | Meaning |
|--------|---------|
| Reproducible | Inputs are included or generated by repository scripts; config and run script exist |
| Partial | Script/config exist, but raw external data or preprocessing is not fully included |
| Synthetic only | Scientifically useful controlled test, but not real-data validation |

## Scorecard

| Benchmark | Status | Inputs Accessible | Assumptions Documented | Outputs Recreated | Notes |
|-----------|--------|-------------------|------------------------|-------------------|-------|
| Analytical validation | Synthetic only | Yes | Yes | Yes | Controlled hydrodynamic sanity tests |
| Historical Rann approximation | Synthetic only | Yes | Yes | Yes | Behavior approximation, not real Gadkabet DEM |

| Literature behavior table | Reproducible | Yes | Yes | Yes | Qualitative behavior-class comparison |
| Remote sensing proxy comparison | Reproducible | Yes | Yes | Yes | Proxy comparison only; not raw SAR/NDWI validation |

## Current Reproducibility Limits

- Phase 6 does not yet include scripts to download FABDEM, Copernicus DEM, SRTM, HydroSHEDS, Sentinel-1, NDWI, Copernicus EMS, or JRC datasets.
- The packaged terrain arrays are small derived artifacts for reproducible benchmarking, not authoritative open-data redistribution.
- Remote-sensing comparison currently uses a proxy mask from local historical outputs.
- Reproduction validates workflow behavior, not calibrated flood forecasting accuracy.

## Gate Assessment

Reproducibility gate status: **PASS WITH LIMITATIONS**

Rationale:

- All Phase 6 scripts are runnable from repository-local data.
- Inputs and assumptions are documented.
- Scientific confidence is conservative.
- Real external open-data ingestion remains future work, not a Phase 6 claim.
