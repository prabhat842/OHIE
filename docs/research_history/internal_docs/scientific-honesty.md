# Scientific Honesty Statement

OHIE currently cannot do the following:

- It does not include groundwater coupling.
- It does not include a calibrated pipe/storm-drain network solver.
- It does not include full shallow-water equations in the new package yet.
- It does not include production-grade outfall submergence and reverse-flow controls.
- It does not include SAR calibration workflows.
- It does not include sediment, erosion, debris blockage, water quality, or salinity-viscosity physics.
- It does not reproduce Yamuna, Rann/Gadkabet, or Gorakhpur historical results exactly.
- It is not a replacement for HEC-RAS, MIKE, LISFLOOD-FP, or other certified tools in final design.

What OHIE can currently support:

- controlled synthetic hydrodynamic validation,
- intervention-aware scenario experiments,
- mass-balance checks,
- routing-method comparison,
- temporal persistence/exposure metrics,
- reproducible YAML/script-based experiments,
- transparent extension by researchers.

The correct claim is:

> OHIE is becoming a transparent, reproducible, intervention-aware hydrodynamic research framework.

The incorrect claim is:

> OHIE is already a calibrated operational flood model.
