# Contributing to OHIE

OHIE is a reproducibility-first scientific scaffold.

## What to contribute

- benchmark cases
- clearer documentation
- limitation tracking
- reproducible examples
- evidence-backed improvements

## What we expect

New claims should come with:

- evidence
- benchmark
- limitation

If a contribution changes scientific meaning, it must also update the relevant docs and reviewer-facing package.

## Rules

- Do not hide tuning behind general language.
- Do not upgrade a proxy comparison into validation.
- Do not claim calibrated hydraulics without calibration.
- Do not add new capabilities before the current limitations are documented.

## Good contribution shape

1. reproduce an existing benchmark
2. document a limitation more clearly
3. add a small test
4. keep the vocabulary inside the lock

## Review standard

Changes are reviewed for:

- reproducibility
- evidence traceability
- scientific restraint
- clarity of limits
