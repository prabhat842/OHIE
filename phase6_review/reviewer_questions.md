# Phase 6 Reviewer Questions

These are the questions that should be answered before Phase 7 begins.

## Scientific Gate

1. Are the real-terrain outputs physically plausible enough for a first benchmark scaffold?
2. Are any mass-balance errors or stage-boundary volume changes suspicious?
3. Are the confidence labels conservative enough?
4. Are the limitations visible enough to prevent accidental overclaiming?

## Reproducibility Gate

1. Can a new researcher run the documented scripts without hidden setup?
2. Are the packaged data sources documented clearly enough?
3. Is it acceptable that external open-data downloads are not yet automated?
4. Should the next revision add raw-data acquisition scripts before any Phase 7 work?

## Researcher Adoption Gate

1. Is `docs/quickstart.md` enough for a first experiment?
2. Are benchmark folders organized in an intuitive way?
3. Is it obvious how to modify forcing, terrain, or interventions?

## Credibility Gate

1. Does the report avoid implying calibrated validation?
2. Is the remote-sensing comparison clearly marked as proxy only?
3. Is the phrase "real terrain" sufficiently qualified as local historical-project-derived terrain?
4. Should the review gate pass, or should Phase 6 enter a revision sprint focused on real external open-data ingestion?
